# EngVocabular — прогресс и устройство

## Как работает приложение

Словарь английских слов с интервальным повторением. Приложение — статический HTML-файл без сервера.

### Основные файлы

| Файл | Назначение |
|------|-----------|
| `index.html` | Главная страница + вся логика (JS) |
| `style.css` | Внешний вид |
| `dictionary.js` | Массив WORDS — список слов (1830 шт.) |
| `data/progress.json` | Прогресс всех пользователей (в репозитории) |
| `data/config.json` | Зашифрованный GitHub-токен владельца (в репозитории) |

## Синхронизация между устройствами

### Как это работает

Прогресс (известные слова) хранится в двух местах:

1. **localStorage браузера** — для быстрого доступа (кеш)
2. **`data/progress.json` в репозитории GitHub** — для синхронизации

Файл `data/progress.json` содержит:
```json
{
  "oleksandr": [1, 5, 23, 45],
  "другой_ник": [2, 8, 15]
}
```

### Шифрование токена

Чтобы на новом устройстве не вводить токен заново, он шифруется никнеймом и сохраняется в `data/config.json`:

```json
{
  "encrypted": "...",
  "salt": "...",
  "iv": "..."
}
```

Расшифровать без никнейма невозможно.

### Сценарии работы

**Вы (владелец) — первое устройство:**
1. Открываете страницу → жмёте «Настройки»
2. Вводите: никнейм + GitHub-токен + owner + repo
3. Токен шифруется → сохраняется в `data/config.json`
4. Прогресс сохраняется в `data/progress.json`

**Вы — новое устройство:**
1. Открываете страницу → жмёте «Настройки»
2. Приложение проверяет, есть ли `config.json` в репозитории
3. Если есть → показывает только поле для никнейма + «Войти»
4. Вводите никнейм → токен расшифровывается → прогресс синхронизируется

**Друг / новый пользователь:**
1. Открывает страницу → видит обычное приложение без окон
2. Прогресс хранится локально в его браузере
3. Чтобы включить синхронизацию — жмёт «Настройки» и настраивает сам

## Список изменений

### 1. GitHub-синхронизация (взамен Firebase)
- Убран Firebase SDK (лишняя регистрация)
- Добавлена синхронизация через GitHub API
- Прогресс хранится в `data/progress.json` в репозитории
- Чтение через `raw.githubusercontent.com` (без авторизации)
- Запись через API с GitHub-токеном

### 2. Шифрование токена (AES-256-GCM)
- Токен шифруется никнеймом и хранится в `data/config.json`
- На новом устройстве нужен только никнейм
- Используется Web Crypto API (PBKDF2 + AES-GCM, встроено в браузер)

### 3. Два режима при входе в настройки
- **Login mode**: только поле для никнейма (когда config.json уже существует)
- **Setup mode**: все поля (никнейм + токен + owner + repo)

### 4. Обработка ошибок
- Кнопка «Отмена» в окне настроек
- Если данные не изменились → config.json не перезаписывается
- Новым пользователям не показывается окно входа (только локальное использование)
